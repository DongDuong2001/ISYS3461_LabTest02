# Security Policy

## Supported Versions

Since this repository serves as a post-assessment reference solution and educational demonstration, it is not actively deployed to production. Security updates are provided on a best-effort basis, primarily targeting the current state of the main branch.

| Version | Supported |
| ------- | --------- |
| Reference Solution (main branch) | :white_check_mark: |
| Custom forks / other branches | :x: |

## Reporting a Vulnerability

We take the security of our tools and code seriously. Since this repository is for educational reference, we ask that you report security issues in a responsible manner:

1. **Non-Critical / Dependency Vulnerabilities:** If the issue is related to dependency updates or minor bugs, please open a standard GitHub Issue.
2. **Critical Vulnerabilities:** If you discover a critical security vulnerability (such as exposed credentials, API key leaks, or critical exploits in the backend gateway or services), please do **NOT** open a public issue. Instead, report it privately to the repository owner or maintainers via the contact email listed on the owner's profile.

We will endeavor to review and address all reported security concerns as quickly as possible. Thank you for helping keep this educational resource safe!
